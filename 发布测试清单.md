# 发布测试清单

用于每次修复、打包、发布前的回归检查。原则：

- 一次只确认一个主题是否回归。
- 不能真实验证的项目要明确标记为“需要补素材”或“需要真机验证”。
- 涉及系统鼠标、注册表、自启动、托盘、安装包的项目，默认需要真机验证。

## 本轮缺陷回归总表

### 1. 导入与匹配稳定性

- [ ] `.cur` 单文件导入正常。
- [ ] 普通 `.ani` 单文件导入后可以预览动画。
- [ ] 无法拆帧的纯 `.ani` 仍能显示首帧或占位图，不出现空白。
- [ ] `.zip` 鼠标包导入正常。
- [ ] `.rar` 鼠标包导入正常。
- [ ] 加密 `.rar` 不闪退，提示“暂不支持加密压缩包”。
- [ ] 没有安装 WinRAR 的干净系统也能导入普通 `.rar`。
- [ ] `.7z` 鼠标包导入正常。
- [ ] 自解压或 PyInstaller `.exe` 鼠标包导入正常。
- [ ] 文件夹导入正常。
- [ ] 单个压缩包内包含多个 `.inf` 时可以批量导入。
- [ ] 导入失败时错误会写入 `错误记录.txt`，UI 不崩溃。
- [ ] `pen / nwpen` 正确匹配到手写。
- [ ] `cross` 正确匹配到精确选择。
- [ ] `hand / link` 正确匹配到链接选择，不误匹配到手写。
- [ ] `pin` 正确匹配到位置选择。
- [ ] `person` 正确匹配到个人选择。
- [ ] 没有 `Pin`、`Person` 资源的老主题包仍能正常导入。

### 2. 方案编辑交互

- [ ] 替换鼠标指针区域支持直接拖入 `.cur`。
- [ ] 替换鼠标指针区域支持直接拖入 `.ani`。
- [ ] 替换鼠标指针区域支持直接拖入图片。
- [ ] 将压缩包拖到资源导入区域时会导入新方案，而不是误当作单个指针。
- [ ] 将单个文件拖到指定角色时，只替换该角色。
- [ ] 角色行拖入错误类型时仅状态提示，不崩溃。
- [ ] 方案列表图标悬停时有明确 hover 反馈。
- [ ] 方案列表图标悬停时出现小 `X`。
- [ ] 点击小 `X` 只移除当前角色，不影响其他角色。
- [ ] 图标 hover 轻微晃动不导致明显卡顿。
- [ ] `.ani` 显示“动”标识。
- [ ] `.cur` 显示“静”标识。
- [ ] 图片和未知格式不显示动静标识。

### 3. 切换状态与计时切换

- [ ] 手动应用方案后，“当前方案”实时刷新。
- [ ] 从资源库应用方案后，“当前方案”实时刷新。
- [ ] 时间切换触发后，“当前方案”实时刷新。
- [ ] 星期切换触发后，“当前方案”实时刷新。
- [ ] 计时切换触发后，“当前方案”实时刷新。
- [ ] 托盘菜单中的“当前配置”与实际一致。
- [ ] “下次切换”能显示时间切换信息。
- [ ] “下次切换”能显示星期切换信息。
- [ ] “下次切换”能显示计时切换信息。
- [ ] 计时切换顺序模式只在选中的方案集合内循环。
- [ ] 计时切换随机模式不会选到未勾选方案。
- [ ] 调整拖拽顺序后，下一个顺序切换按新顺序执行。
- [ ] 重启程序后计时切换配置完整恢复。

### 4. 打包、安装包与发布链路

- [ ] 绿色版可正常启动。
- [ ] 完整版可正常启动。
- [ ] 绿色版生成安装包时不会提示找不到带 PyInstaller 的 Python。
- [ ] 若当前版本不包含安装包运行时，会明确提示下载完整版。
- [ ] 生成安装包过程不弹黑色控制台窗口。
- [ ] 打包失败时能查看 `pyinstaller_error.log`。
- [ ] `runtime` 已被正确打入对应包体。
- [ ] README 下载链接与最新 Release 资产一致。
- [ ] 网页下载链接与最新 Release 资产一致。
- [ ] `dist` 与 `release-assets` 的目标产物哈希一致。
- [ ] 当前对外声明仅支持 Windows x64，没有误导为 arm64 可用。

### 5. 启动、自启动、托盘与后台

- [ ] Win10 普通用户权限下正常启动。
- [ ] Win11 普通用户权限下正常启动。
- [ ] 冷启动到主窗口可见的耗时可接受。
- [ ] 第二次启动时会唤起已有窗口，而不是打开第二个完整界面。
- [ ] 关闭窗口后，如设置为保留托盘，托盘图标仍存在。
- [ ] 托盘左键可打开主窗口。
- [ ] 托盘右键“打开”可打开主窗口。
- [ ] 托盘右键“退出”后进程完全退出。
- [ ] 开启“隐藏任务栏图标”后，关闭窗口不显示托盘，只保留后台。
- [ ] 开启“隐藏任务栏图标”后，后台切换仍正常工作。
- [ ] 设置页显示的自启动状态与系统实际状态一致。
- [ ] 诊断信息能显示 Run 项、启动文件夹、任务计划、托盘 PID、后台 PID、进程路径。
- [ ] “立即测试后台启动”会按当前设置启动 `--tray` 或 `--background`。
- [ ] stale pid 文件不会导致状态误判。

## 最小手测路径

### 导入

- [ ] 用 1 个 `.cur`、1 个普通 `.ani`、1 个无法拆帧的 `.ani` 完成三次单文件导入。
- [ ] 用 1 个 `.zip`、1 个 `.rar`、1 个 `.7z`、1 个 `.exe`、1 个文件夹完成五次导入。
- [ ] 用 1 个包含多个 `.inf` 的压缩包确认批量导入。

### 角色匹配

- [ ] 用包含 `pen / cross / hand / pin / person` 命名的主题包导入并检查落位。
- [ ] 用 1 个老主题包确认缺少 `Pin / Person` 时仍可导入。

### 切换

- [ ] 分别验证时间切换、星期切换、计时顺序切换、计时随机切换。
- [ ] 每次切换后确认当前方案、下次切换、托盘菜单三处同步刷新。

### 打包

- [ ] 在开发机验证绿色版和完整版打包。
- [ ] 在无 Python 的干净环境验证绿色版生成安装包，或验证清晰降级提示。

### 自启动

- [ ] 开启自启动并重启系统，确认实际启动方式和设置一致。
- [ ] 分别验证“保留托盘”和“隐藏任务栏图标”两种关闭窗口后的行为。
