# QA 与发布检查规格

## 自动检查

- `python -m compileall main.py fluent_ui.py`
- 纯函数单元测试：文件类型标识、资源汇总、角色匹配、INF 解析、切换状态计算。
- 打包前检查暂存区，确认不包含 WinUI 改动和误删素材。

## 手动 UI 检查

- 资源库列表视图：单选、应用、删除、刷新后状态正确。
- 资源库宫格视图：选择和底部操作与列表一致。
- 方案编辑页：`.ani` 和 `.cur` 标识不遮挡图标。
- 替换资源池：单文件拖入、删除、拖到角色行都正常。
- 资源库预览：小图不显示逐个动静标识，方案名旁只显示汇总。

## 真机检查

- Windows 10 x64 与 Windows 11 x64。
- 普通启动、后台启动、托盘启动。
- 开机自启动、关闭窗口后托盘/后台保留。
- 应用方案、恢复应用前方案、系统鼠标大小。
- 生成绿色版和安装器后从干净目录启动。

## 记录要求

- 每次发布记录 Windows 版本、是否管理员、是否安装安全软件。
- 真实 `.ani`、加密 RAR、多 INF 包、中文路径包必须记录样本来源。
- 未拿到真实素材的项目只能标记为待验证，不能标记通过。
