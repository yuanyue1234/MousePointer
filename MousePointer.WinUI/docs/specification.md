# 规格说明

## 目标用户

- 新手用户：用图形界面导入、预览、应用和恢复鼠标指针方案。
- 鼠标指针制作者：快速整理资源包、修正角色落位、生成可分发安装器。

## 核心对象

- 鼠标角色：Windows `Control Panel\Cursors` 下的 17 个注册表项。
- 方案：一个目录和一份 `scheme.json`，记录方案名称、保存时间、角色到文件的映射、可选鼠标大小。
- 资源包：压缩包、安装器或文件夹，导入后转成一个或多个方案。
- 切换规则：固定时间、星期、计时顺序/随机。

## 功能规格

### 方案管理

- 用户可以新建、重命名、删除方案。
- 用户可以为任意角色选择 `.cur`、`.ani` 或图片素材。
- 用户可以只配置部分角色并应用；未配置角色不写入。
- 保存方案时，素材复制到方案目录，`scheme.json` 是唯一元数据来源。

### 导入

- 支持文件夹、`.zip`、`.rar`、`.7z` 和 `.exe`。
- 导入目录存在 INF 时，优先按 INF alias 和注册表写法落位。
- 没有 INF 时，按文件名关键词匹配，最后按 `01.cur` 到 `17.cur` 编号兜底。
- 一个资源包内多份 INF 会导入为多个方案。
- RAR 失败时需要给出可理解原因，不允许静默失败。

### 应用和恢复

- 应用前先备份当前系统鼠标方案到 `cursor_backup.json`。
- 应用时写入 `HKCU\Control Panel\Cursors`，并调用 Windows API 刷新。
- 鼠标大小写入 `CursorBaseSize` 和 `Software\Microsoft\Accessibility\CursorSize`。
- 恢复按钮从备份文件恢复注册表。

### 自动切换

- 固定时间：每天到达 `HH:mm` 后应用指定方案。
- 星期切换：每天首次检测到当天规则时应用。
- 计时切换：按秒/分钟间隔触发，可在选中方案集合内顺序或随机。
- `--background` 不显示窗口，只执行规则。
- `--tray` 显示托盘入口，支持打开和退出。

### 设置

- 存储目录、输出目录、GitHub URL 可配置。
- 自启动写入 HKCU Run 项，按设置选择 `--background` 或 `--tray`。
- 文件关联写入 HKCU Classes，`.cur` / `.ani` 打开到轻量预览入口。
- 诊断信息应包含路径、自启动、文件关联、当前方案和下次切换。

## 非功能规格

- 架构：Core 不依赖 WinUI；WinUI 只依赖 Core。
- 数据：JSON 使用 UTF-8，并保留中文可读文本。
- 平台：先支持 Windows x64，不承诺未验证的 ARM64。
- 安全：默认只写 HKCU，不要求管理员权限。
- 可靠性：系统级操作失败必须写入 `错误记录.txt`。
