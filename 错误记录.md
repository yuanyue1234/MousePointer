# 错误记录

程序运行、生成安装器、WinRAR 封装或定时切换失败时，会把错误追加到这里。

## 2026-04-13 修复记录

```text
问题：
1. 鼠标文件默认存放在 LocalAppData，存在误删和迁移不清晰的问题。
2. 新增资源包不应直接变成 03/04 默认方案，需要作为资源库方案按需添加。
3. 资源库下载和导入流程不够明确。
4. 设置入口缺失，无法修改鼠标文件存放位置。

处理：
1. 默认存储目录改为 %APPDATA%\MouseCursorThemeBuilder\mouse_files。
2. 默认方案恢复为 01方案、02方案；其他内置压缩包通过资源库刷新导入。
3. 添加资源库页、设置页、存储目录设置、资源刷新扫描。
4. Edge App 模式打开在线资源库时尝试指定下载目录为资源库目录。
```

## 2026-04-13 原框架继续修复

```text
问题：
1. 表格行虽然放进 Canvas，但内部列宽没有完全跟随可用宽度变化。
2. 生成安装包仍优先使用 WinRAR，且保存位置固定，不符合“选择保存位置后打开文件夹”的流程。
3. 应用和生成安装包是同步执行，界面容易看起来卡死。
4. 设置里缺少安装包默认保存位置。

处理：
1. 保留 Tk 原框架，不再迁移 Qt。
2. 调整鼠标方案列表 Canvas 与行内 grid 权重，让文件列随窗口宽度变化。
3. 生成安装包改为只使用 Python/PyInstaller，生成前选择保存目录，生成后打开对应文件夹。
4. 应用和生成安装包改为后台线程执行，并显示等待窗口。
5. 设置页增加安装包默认保存位置，默认位于 %APPDATA%\MouseCursorThemeBuilder\installers。
6. 应用 .cur/.ani 方案时直接写入系统方案，减少不必要的重复转换和复制。
7. 已重新生成主程序：dist\鼠标配置生成器.exe，启动探针通过。
```

## 2026-04-13 后台保留与资源导入修复

```text
问题：
1. 勾选自启动后只写入开机启动，没有在手动关闭窗口时保留后台，时间/星期切换依赖后台调度不够明确。
2. 时间/星期方案立即应用时没有等待窗口。
3. 资源库三个按钮是纵向排列，不符合横排对齐。
4. 拖入资源包缺少直接导入为新方案的入口。
5. 生成的安装包没有使用所选鼠标指针图像作为图标。

处理：
1. 勾选“自启动并保留后台”后，关闭窗口会隐藏界面并继续运行调度线程。
2. 时间切换、星期切换应用走居中等待窗口。
3. 资源库按钮改为横排显示。
4. 资源库页支持拖入 zip/rar/7z/exe，识别后保存为新方案。
5. 生成安装包时优先用“正常选择”鼠标图像生成 exe icon，并继续使用 --noconsole。
```

## 2026-04-13 后台与 EXE 导入复测修复

```text
问题：
1. 关闭主窗口后只是隐藏 GUI，不够稳定，用户侧观察为后台没有保留。
2. 拖入由本工具生成的 PyInstaller 单文件 EXE 时，会被当作普通压缩包处理并提示不是可解压程序。

处理：
1. 关闭窗口且勾选“自启动并保留后台”时，改为启动隐藏的 --background 后台进程，再退出 GUI。
2. 后台进程增加 pid 锁，避免重复保留后台。
3. EXE 导入增加 PyInstaller CArchive 解析，能抽取 assets 里的 .cur/.ani 并添加为新方案。
4. 在鼠标方案表格行拖入 zip/rar/7z/exe 时，也会按资源包导入为新方案。
5. 已重新生成主程序：dist\鼠标配置生成器.exe，启动探针通过。
```

## 2026-04-14 托盘后台与资源库预览

```text
问题：
1. 关闭窗口后的后台需要有托盘图标，并能通过右键“退出”结束。
2. 开启自启动后缺少可见的系统通知提示。
3. 资源库下方没有展示已有资源。
4. 资源库按钮文案需要从“刷新并添加方案”改为“刷新”。

2. 开启自启动时通过托盘通知提示“启动选项已更改”。
3. 资源库页下方新增已有资源预览区，按方案展示鼠标图标，动态 .ani 会循环显示。
4. 调整页面背景为淡蓝色，并将资源库三个操作按钮横向显示。
```

## 2026-04-14 资源库滚动与鼠标大小预览

```text
问题：
1. 资源库已有资源区只显示部分方案，新增方案后不能稳定实时刷新。
2. 已有资源区没有横向滑动和纵向滚动，图标较多时显示不全。
3. 实时预览缺少鼠标大小调节入口。

处理：
1. 资源库改为读取方案库全部方案，刷新或拖入导入后会重新渲染已有资源区。
2. 已有资源区增加纵向滚动；每个方案图标条支持 Shift+滚轮和鼠标拖拽横向查看，不显示横向滚动条。
3. 实时预览上方新增鼠标大小滑条，范围 1-15，对应 32px 到 256px；超出推荐范围 2-7 时滑条变红并提示。
4. 勾选“实时更新鼠标大小”后，预览和生成游标会使用当前大小；未勾选时使用默认 7。
```

## 2026-04-14 系统鼠标指针大小写入

```text
问题：
1. 鼠标大小此前只影响预览和生成，没有直接更改 Windows 系统指针大小。
2. 用户要求应用顺序为：刷新鼠标参数 -> 更改大小 -> 应用指针方案。

处理：
1. 增加 CursorBaseSize 写入：HKCU\Control Panel\Cursors\CursorBaseSize。
2. 应用方案、时间/星期切换、后台切换、生成的安装器都改为先刷新鼠标参数，再写入指针大小，再应用指针方案。
3. 鼠标大小等级 1-15 映射为 32px-256px。
```

## 2026-04-14 鼠标大小应用值修复

```text
问题：
1. 未勾选实时更新时，应用系统鼠标大小仍回落到默认值，导致最终大小看起来固定。
2. 操作按钮顺序需要调整。

处理：
1. 系统 CursorBaseSize 写入始终读取滑条当前值，不再受“实时更新鼠标大小”勾选状态影响。
2. 底部按钮改为：清除、应用、生成安装包、鼠标大小设置。
```

## 2026-04-14 清除默认与托盘打开

```text
问题：
1. 清除按钮误解为清空方案配置，实际需要恢复 Windows 默认鼠标指针。
2. 时间切换和星期切换默认不应自动填写当前方案。
3. 托盘图标需要右键“打开”，并支持左键直接打开窗口。
4. 鼠标大小写入后仍需更强刷新。

处理：
1. 清除按钮改为写回 Windows 默认鼠标指针空方案并刷新。
2. 时间切换和星期切换方案下拉默认空白。
3. 托盘菜单新增“打开”，并将其设为默认动作，左键可打开窗口。
4. 写 CursorBaseSize 后调用 SystemParametersInfoW(SPI_SETCURSORS) 并传入像素值，再应用指针方案。
```
## 2026-04-14 取消鼠标大小写入与资源库滚动调整

```text
问题：
1. 直接写入 Windows 鼠标大小在不同系统上表现不稳定，用户决定取消该能力。
2. “清除”按钮不再需要，避免和方案清空、系统默认恢复产生误解。
3. 生成安装包不应携带鼠标大小配置。
4. 资源库“已有资源”区域单独滚动不顺滑，和整页滚动体验割裂。

处理：
1. 应用方案改回只刷新鼠标参数并应用指针方案，不再写 CursorBaseSize。
2. 主操作区移除清除按钮，把“鼠标大小设置”放到第一位，并添加提示“更改鼠标至对应大小后应用方案”。
3. 安装包生成源码移除鼠标大小字段和大小写入逻辑。
4. 资源库已有资源改为跟随资源库页面整体滚动；每个方案的图标条仍保留横向拖动和 Shift+滚轮查看。
```
